# Build Plan

## Unit 01: Foundation

Build Next.js, TypeScript, Tailwind, basic config, package scripts, and folder structure.

Done when:

- Project installs with npm.
- TypeScript is strict.
- Tailwind compiles.
- Root routes exist.

## Unit 02: Product Shell And Login

Build role-based mock login, persistent sidebar, operating mode context, and readiness strip.

Done when:

- `/login` is usable.
- App shell wraps operational pages.
- Navigation reaches all core routes.

## Unit 03: Portfolio And Readiness

Build portfolio command center and Data & Policies hub.

Done when:

- `/portfolio` shows portfolio health, buckets, active hands, and next actions.
- `/data` shows data sources, policies, validation, blockers, and readiness states.

## Unit 04: Coworker And Hand Drafting

Build AI coworker workspace and Banking Hand list/detail.

Done when:

- `/workspace` shows conversation plus artifacts.
- `/hands` lists hands.
- `/hands/h1` shows lifecycle, gates, and outbound preview.

## Unit 05: Governance, Runs, And Audit

Build approvals, run monitoring, audit timeline, and outcome movement.

Done when:

- `/approvals` shows review queue.
- `/runs` shows active runs.
- `/runs/r1` shows progress, failures, HITL pause, outcomes, and immutable audit.

## Unit 06: Customers, Settings, And Docs

Build supporting customer/admin views and complete documentation.

Done when:

- `/customers` and `/customers/c1` are usable.
- `/settings` is usable.
- README and system design docs cover the challenge checklist.
- `npm run typecheck` and `npm run build` pass.
